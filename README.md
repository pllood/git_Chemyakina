1. ВВЕДЕНИЕ
Настоящее техническое задание распространяется на разработку программного обеспечения для изучения ключевых разделов географии. В курсе географии темы, связанные с политическим устройством мира, физическими особенностями Земли являются одними из самых фундаментальных, но часто вызывают сложности у учащихся из-за большого объема информации и необходимости ее систематизации. Усвоение таких данных, как расположение стран и столиц, физико-географических объектов и демографической статистики, требует от учащихся развитой памяти и умения работать с картографическим материалом.
Существующие программные решения, позволяющие изучать географию, часто являются либо узкоспециализированными, предназначенными для профессиональных картографов и аналитиков, либо же, наоборот, слишком упрощенными, не дающими полного представления о взаимосвязях между различными географическими аспектами. Примером таких программ могут быть интерактивные карты, которые предоставляют скорее статичный справочный материал, чем комплексную обучающую систему, или же профессиональные геоинформационные системы (ГИС), которые имеют сложные пользовательские интерфейсы, требующие специальных знаний.
Разрабатываемая программа позволит наглядно изучить политическую карту мира, физическую географию материков и основы демографии, а также проверить и закрепить свои знания по каждой теме с помощью специализированных тренажеров и тестов.

2. ОСНОВАНИЯ ДЛЯ РАЗРАБОТКИ
Программа разрабатывается на основе учебного плана кафедры программы подготовки специалистов среднего звена по специальности среднего профессионального образования 09.02.07 Информационные системы и программирование.

3. НАЗНАЧЕНИЕ
Программа предназначена для интерактивного изучения и визуализации ключевых разделов физической и социально-экономической географии. Система обеспечивает систематизированное представление географических объектов, явлений и процессов и предоставляет инструменты для их изучения и закрепления, ориентированные на пользователей разного уровня подготовки, от школьников до любителей географии.
4. ТРЕБОВАНИЯ К ПРОГРАММЕ ИЛИ ПРОГРАММНОМУ ИЗДЕЛИЮ 
4.1.Требования к функциональным характеристикам
4.1.1.Программа должна обеспечивать интерактивное изучение тем
4.1.1.1. Должна иметь возможность выбора одной из шести тем для изучения: океаны, реки, горные породы, минералы, страны, стихийные бедствия
4.1.1.2. Каждая тема должна разбиваться на несколько лекций с теоретическим материалом
4.1.1.2.1. Для каждой лекции должен быть представлена структурированная информация с подходящими изображениями
4.1.1.3. Для каждой темы должен быть представлен  интерактивный тренажёр для закрепления материала
4.1.1.3.1. В тренажере должна быть реализована возможность корректности выполнения
4.1.1.4.2. Задания для тренажера должны выбираться случайным образом из банка возможных вариантов, исключая повторения
4.1.1.4. Для каждой темы должен быть представлен  итоговый тест из 10 вопросов для проверки знаний
4.1.1.4.1. По завершению тестирования должно выводится количество заработанных баллов и оценка по пятибалльной системе
4.1.1.4.2. Вопросы для тестирования должны выбираться случайным образом из банка вопросов, исключая повторения
4.1.1.5. Должна быть возможность просмотра статистики пользователя 
4.1.2.Программа должна иметь интерфейс
4.1.2.1. Интерфейс должен быть интуитивно понятным и простым в использовании
4.1.4.2. Основные элементы управления должны быть легко доступны 
4.1.4.3. Цветовая схема должна быть приятной для глаз и соответствовать тематике программы
4.1.4.4. Элементы управления должны быть четко видимы и легко различимы
                       4.1.3. Программа должна обеспечивать систему аутентификации и регистрации пользователей
4.1.3.1. Должна быть реализована возможность входа в программу (аутентификации) по логину и паролю.
4.1.3.2. Должна быть реализована возможность регистрации нового пользователя через ввод адреса электронной почты, логина и пароля.
4.1.3.3. Система должна проверять введенные данные при регистрации на соответствие следующим критериям:
4.1.3.3.1. Все поля (электронная почта, логин, пароль) обязательны для заполнения.
4.1.3.3.2. Данные во всех полях должны состоять только из разрешенных латинских символов, цифр и специальных знаков (@, ., -).
4.1.3.3.3. Логин и пароль должны содержать не менее 6 и не более 10 символов.
4.1.3.3.4. Адрес электронной почты должен соответствовать стандартному формату (наличие символа @ и домена).
4.1.3.3.5. Электронная почта и логин должны быть уникальными в системе. Программа должна проверять отсутствие совпадений с уже зарегистрированными пользователями и информировать о нарушении уникальности.
4.1.3.4. Учётные данные зарегистрированных пользователей должны сохраняться в защищенном хранилище (например, в файле в формате JSON).
4.1.3.5. После успешной аутентификации или регистрации программа должна запоминать текущего пользователя для последующего ведения его персональной статистики.
4.1.3.6. Должен быть предусмотрен удобный переход между окнами регистрации и аутентификации.
